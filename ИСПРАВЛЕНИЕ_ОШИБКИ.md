# ⚠️ Исправление ошибки 404 на Vercel

Если вы получили ошибку `DEPLOYMENT_NOT_FOUND`, это значит что при развертывании что-то пошло не так. Не волнуйтесь, это легко исправить!

## ✅ Решение

### Способ 1: Пересоздать проект (РЕКОМЕНДУЕТСЯ)

1. **Удалите предыдущее развертывание на Vercel**
   - Откройте https://vercel.com/dashboard
   - Найдите проект `border-queue-monitor`
   - Нажмите на него → Settings → Danger Zone → Delete Project

2. **Очистите GitHub репозиторий**
   ```bash
   git status
   rm -rf node_modules
   rm package-lock.json
   git add .
   git commit -m "Clean up for redeployment"
   git push
   ```

3. **Пересоздайте Vercel проект**
   - Откройте https://vercel.com/dashboard
   - Нажмите "Add New Project"
   - Выберите `border-queue-monitor` с GitHub
   - **ВАЖНО:** На экране Import Settings:
     - Framework: выберите **"Vite"**
     - Build Command: `npm run build`
     - Output Directory: `dist`
     - Install Command: `npm install`
   - Нажмите "Deploy"

4. **Дождитесь завершения сборки**
   - На странице проекта будет прогресс-бар
   - Когда закончится, кликните "Visit" → откроется ваше приложение

---

### Способ 2: Через Vercel CLI (если первый способ не помог)

```bash
# Установите Vercel CLI
npm install -g vercel

# Логинитесь
vercel login

# Разверните (откажитесь от всех вопросов кроме последних)
vercel --prod

# Следуйте подсказкам
# Выберите "Create and deploy" when asked
# Выберите Node.js runtime
```

## 🔍 Что проверить перед развертыванием

### 1. Структура папок
```
border-queue-monitor/
├── src/
│   ├── App.jsx ✓
│   ├── main.jsx ✓
│   └── index.css ✓
├── index.html ✓
├── package.json ✓
├── vite.config.js ✓
└── tailwind.config.js ✓
```

### 2. package.json должен содержать
```json
{
  "scripts": {
    "build": "vite build",
    "dev": "vite"
  },
  "type": "module"
}
```

### 3. Сборка локально работает?
```bash
npm install
npm run build
# Должна создаться папка dist/
ls dist/
```

Если сборка падает локально, проверьте:
- Версию Node.js: `node --version` (должно быть 18+)
- Ошибки при `npm install`

## 🆘 Если ничего не помогло

### Проверьте логи Vercel

1. Откройте ваш проект на Vercel
2. Перейдите в "Deployments"
3. Кликните на последний деплой
4. Откройте "Logs"
5. Ищите красные ошибки

### Типичные ошибки и решения

#### "Module not found"
```
Решение: rm -rf node_modules && npm install
```

#### "Cannot find vite"
```
Решение: Убедитесь что vite в devDependencies в package.json
```

#### "React not found"
```
Решение: npm install react react-dom
```

#### Timeout при установке
```
Решение: Vercel может иметь проблемы. Попробуйте:
1. Дождитесь 5 минут
2. Нажмите "Redeploy" на Vercel
```

## 📝 Альтернатива: Netlify

Если Vercel продолжает вызывать проблемы, используйте **Netlify** (еще проще!):

```bash
# Установите Netlify CLI
npm install -g netlify-cli

# Сначала соберите локально
npm run build

# Разверните
netlify deploy --prod --dir=dist

# Первый раз спросит что-то - просто следуйте подсказкам
```

## ✨ Проверка что все работает

После успешного развертывания:

1. Откройте ваш URL (например `https://your-project.vercel.app`)
2. Должны загрузиться КПП в списке
3. Нажмите на один из них
4. Должны загрузиться данные очереди

Если данные не загружаются - откройте Console (F12) и посмотрите ошибки.

## 🚨 Последний вариант: Локальное тестирование

Если вы хотите сначала убедиться что все работает локально:

```bash
# Установите все
npm install

# Запустите dev сервер
npm run dev

# Откройте http://localhost:5173
# Должно работать без ошибок

# Проверьте сборку
npm run build
# Должна появиться папка dist/

# Проверьте production версию
npm run preview
# Откройте показанный URL
```

Если все работает локально, то на Vercel должно работать тоже!
