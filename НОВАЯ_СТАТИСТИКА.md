# 📊 Новая статистика выезда машин

В приложение добавлена система отслеживания статистики выезда автомобилей!

## Что показывает статистика?

### 📋 Блок статистики находится:
**Над таблицей очереди**, под кнопкой "Обновить"

### 📊 Три основных метрики:

#### 1️⃣ Всего авто в очереди
- **Показывает:** Сколько машин сейчас в системе (все статусы)
- **Где:** На КПП сейчас находится столько машин
- **Обновляется:** При каждом нажатии "Обновить"

#### 2️⃣ Выехало в течении часа
- **Показывает:** Сколько машин выехало за последний час
- **Как рассчитывается:** Считается количество машин со статусом "Завершен" (выехали) и сравнивается с предыдущими данными
- **Данные:** Сохраняются в браузере (localStorage)
- **Время:** За последний час

#### 3️⃣ Выехало за сутки
- **Показывает:** Сколько машин выехало за последние 24 часа
- **Как рассчитывается:** Собирает данные за все 24 часа
- **Данные:** Сохраняются в браузере (localStorage)
- **Время:** За 24 часа

### 📈 Бонус: Средний темп (за последние 5 часов)
- **Показывает:** Среднее количество машин в час
- **Как рассчитывается:** Берет данные выезда за последние 5 часов и делит на 5
- **Помогает:** Понять скорость обработки в среднем

---

## 📈 Пример статистики:

```
┌─────────────────────────────────────────────────────────┐
│  Всего авто в очереди: 45                               │
│  Выехало в течении часа: 12                             │
│  Выехало за сутки: 177                                  │
│                                                         │
│  Средний темп (за последние 5 часов): 14 авто в час   │
└─────────────────────────────────────────────────────────┘
```

**Интерпретация:**
- 45 машин ждут в очереди
- За последний час выехало 12 машин
- За весь день выехало 177 машин
- В среднем 14 машин в час

## 💾 Как работает сохранение данных?

### Локальное хранилище (localStorage)
Приложение автоматически сохраняет статистику в браузере:
- 📁 Каждый КПП имеет свою историю
- ⏰ Данные хранятся последние 24 часа
- 🔄 Обновляется при каждом нажатии "Обновить"

### Что происходит:
1. Вы открываете приложение
2. Выбираете КПП
3. Нажимаете "Обновить"
4. Приложение считает количество выехавших машин
5. Сохраняет эту информацию в браузере
6. При следующем обновлении сравнивает с предыдущим значением

### Сохранение данных:
```javascript
// Данные сохраняются локально в браузере
// Если закроете браузер - данные сохранятся
// Если перезагрузите страницу - данные восстановятся
// Если очистите кеш браузера - данные удалятся
```

## ❓ Часто задаваемые вопросы

### Q: Почему статистика может быть неточной?
A: Потому что мы считаем машины со статусом "Завершен". Если машина вышла из системы, её статус может не сохраниться. Это оценка на основе доступных данных.

### Q: Что если я закрою браузер?
A: Данные сохранятся в браузере. При открытии вернитесь на сайт - история восстановится.

### Q: Что если я очищу кеш?
A: История удалится. Начнет считать заново с момента очистки.

### Q: Почему данные разные на разных КПП?
A: Потому что каждый КПП имеет свою историю. Для каждого КПП ведется отдельная статистика.

### Q: Сбрасывается ли статистика ночью?
A: Нет, статистика ведется непрерывно. Но хранятся только данные за последние 24 часа.

### Q: Можно ли экспортировать данные?
A: Пока нет, но данные сохраняются локально. Вы можете скопировать их из Developer Tools (F12 → Application → localStorage).

## 🛠️ Техническая информация

### Как приложение считает выезды:
```javascript
// Алгоритм:
1. Получаем текущую очередь через API
2. Считаем машины со статусом 5 (Завершен)
3. Сохраняем это значение + текущее время
4. При следующем обновлении:
   - Сравниваем текущее количество с предыдущим
   - Разница = машины, выехавшие за этот период
5. Для среднего за 5 часов:
   - Берем данные за последние 5 часов
   - Делим на 5
```

### Где хранятся данные:
```
localStorage[`stats_CHECKPOINT_ID`] = [
  { timestamp: 1718282400000, completed: 45 },
  { timestamp: 1718282460000, completed: 46 },
  { timestamp: 1718282520000, completed: 48 },
  ...
]
```
