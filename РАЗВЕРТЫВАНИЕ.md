# 🚀 Развертывание приложения на бесплатный хостинг

## Вариант 1: Vercel (САМЫЙ ПРОСТОЙ) ⭐

### Шаг 1: Загрузка кода на GitHub

1. Перейдите на https://github.com (создайте аккаунт если нет)
2. Нажмите **+ → New repository**
3. Введите имя: `border-queue-monitor`
4. Нажмите **Create repository**
5. В терминале выполните:
```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/YOUR_USERNAME/border-queue-monitor.git
git push -u origin main
```

### Шаг 2: Развертывание на Vercel

1. Перейдите на https://vercel.com/sign-up
2. Нажмите **Continue with GitHub**
3. Авторизуйтесь на GitHub
4. Вернитесь на https://vercel.com/dashboard
5. Нажмите **Add New → Project**
6. Выберите репозиторий `border-queue-monitor`
7. Нажмите **Import**
8. Оставьте настройки по умолчанию
9. Нажмите **Deploy**

**Ждите 1-2 минуты...**

✅ Готово! URL вида `https://border-queue-monitor.vercel.app`

## Вариант 2: Netlify

### Шаг 1: Загрузка на GitHub (как выше)

### Шаг 2: Развертывание на Netlify

1. Перейдите на https://netlify.com/sign-up
2. Нажмите **Sign up with GitHub**
3. Авторизуйтесь
4. На главной странице нажмите **Add new site → Import an existing project**
5. Выберите GitHub
6. Найдите и выберите `border-queue-monitor`
7. Настройки:
   - Build command: `npm run build`
   - Publish directory: `dist`
8. Нажмите **Deploy site**

✅ Готово! URL вида `https://your-site-name.netlify.app`

## Вариант 3: GitHub Pages (бесплатный хостинг GitHub)

### Шаг 1: Загрузка на GitHub (как выше)

### Шаг 2: Активировать GitHub Pages

1. Откройте репозиторий на GitHub
2. Перейдите в **Settings → Pages**
3. Под "Build and deployment" выберите:
   - Source: **GitHub Actions**
4. Нажмите **Create a new workflow yourself**
5. Вставьте этот код:

```yaml
name: Deploy to GitHub Pages

on:
  push:
    branches: [ main ]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
    - uses: actions/checkout@v3
    - uses: actions/setup-node@v3
      with:
        node-version: 18
    - run: npm install
    - run: npm run build
    - uses: peaceiris/actions-gh-pages@v3
      with:
        github_token: ${{ secrets.GITHUB_TOKEN }}
        publish_dir: ./dist
```

6. Нажмите **Commit changes**
7. Дождитесь выполнения workflow (см. вкладка Actions)

✅ Готово! URL: `https://YOUR_USERNAME.github.io/border-queue-monitor`

## Проверка работы

После развертывания:

1. Откройте ссылку на приложение
2. Выберите КПП из списка
3. Должна загрузиться информация о КПП и очередь

Если данные не загружаются:
- Откройте консоль браузера (F12 → Console)
- Проверьте наличие CORS ошибок
- Попробуйте обновить страницу (Ctrl+R)

## Обновление приложения

Чтобы обновить приложение после внесения изменений:

1. Внесите изменения в файлы локально
2. Загрузите на GitHub:
   ```bash
   git add .
   git commit -m "Update: описание изменений"
   git push
   ```
3. Приложение автоматически перезагружается! 🔄

## Если что-то не работает

### Ошибка "npm: command not found"
- Установите Node.js с https://nodejs.org

### Ошибка при git push
- Проверьте, что правильно скопирована ссылка репозитория
- `git remote -v` покажет текущую ссылку
- Если нужно изменить: `git remote set-url origin https://correct-url`

### Приложение не обновляет данные
- Это может быть CORS блокировка
- Попробуйте обновить страницу
- Нажмите кнопку "Обновить" в приложении

### Очередь не загружается
- Проверьте интернет соединение
- Убедитесь, что API работает: https://belarusborder.by/info/monitoring-new?token=test&checkpointId=53d94097-2b34-11ec-8467-ac1f6bf889c0
